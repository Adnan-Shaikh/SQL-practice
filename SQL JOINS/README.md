# SQL Joins Practice

## Overview

This project documents my hands-on practice with SQL JOIN operations.

The goal was to understand how data from multiple tables can be combined and analyzed using different types of joins.

---

## Tables Used

### Employees Table

Contains:

- Employee ID
- Name
- Salary
- Department ID


### Departments Table

Contains:

- Department ID
- Department Name


---

# Concepts Practiced

## INNER JOIN

Used to retrieve only matching records from both tables.

Example:

Employees who have an assigned department.


## LEFT JOIN

Used when we want all records from the left table, even if matching data does not exist.

Example:

Finding employees without departments.


## RIGHT JOIN

Used when we want all records from the right table.

Example:

Showing departments even if they have no employees.


---

# Additional SQL Concepts Practiced

- SELECT
- WHERE
- GROUP BY
- HAVING
- COUNT()
- SUM()
- MAX()
- NULL filtering


---

# Key Learning

The biggest takeaway from JOIN practice was understanding how relational databases connect information stored in different tables.

Instead of storing everything in one large table, databases separate information and JOIN operations help combine it whenever required.
